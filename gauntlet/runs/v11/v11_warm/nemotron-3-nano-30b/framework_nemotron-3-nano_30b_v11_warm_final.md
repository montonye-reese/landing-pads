## 3 CORE INSTRUMENTS (the “operational toolkit”)

| Instrument | What it does | How it works (key mechanism) |
|---|---|---|
| **Planetary Data Registry (PDR)** | Stores *every* metadata point about data collection, processing, processing‑purpose, and provenance. | All metadata is immutable; any change must be signed on a **blockchain‑anchored ledger** that can be queried by any citizen. |
| **Life‑Data Credit (LCD) Token** | Rewards any *verified* act of *voluntary, informed data sharing* **or** any act that *improves* planetary health (e.g., tree‑planting, water‑filtration). | Tokens are minted when a *verified consent token* (from the PDA) is issued; they can be spent on **regenerative‑budget grants** or *saved* as a personal credit score. |
| **Regenerative‑Data Bond** | A bond that only *pays out* when verified restoration projects meet **verified‑impact thresholds** (e.g., restored wetland area > X ha, water‑quality index > X). | The bond’s smart‑contract releases payments only when independent auditors (including citizen panels) verify the target has been met. |
| **Citizen‑Data Assembly** | Rotating, geographic‑based citizen assemblies that evaluate data‑use proposals. | Every 6 months a randomly selected group of 200‑plus citizens votes on *whether a data‑flow proposal may be processed*; its outcome automatically triggers **LDC debits** or **grant awards**. |
| **Cognitive‑Well‑Being Metric & Attentional‑Integrity Review** | Scores an AI‑driven algorithm on how much it perturbs a user’s attention, mood, or decision‑making without consent. | If the score exceeds a threshold, the algorithm is **blocked** from deployment; any misuse *auto‑triggers* an LDC debit and a mandatory *re‑audit* of the algorithm. |
| **Citizen‑Data Trusts / Data‑Trusts** | A community‑owned entity that aggregates data contributed by its members, sells *qualified* aggregations to researchers *only* after a **cultural‑impact audit**, and shares the proceeds back to the community. | Guarantees **cultural‑wealth preservation** and *reciprocal benefit* for marginalized groups. |
| **Cognitive‑Well‑Being Dashboard** | Shows each user (in their PDA) the health of their attention, sleep, stress, and the *risk* of algorithmic manipulation. | The dashboard auto‑updates; any spike triggers an **Alert** that can be used by the user to *block* further algorithmic nudging. |
| **Citizen‑Data‑Demand‑Response System** | When an LCR (Life‑Data Credit) is spent, the system automatically logs the transaction, updates the PDR, and *pays* the token to a **community data‑trust**. | This *links personal consumption directly* to *community benefit* (e.g., funding a local water‑filtration project). |
